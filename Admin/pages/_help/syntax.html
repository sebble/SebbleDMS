Block level elements

Sometimes you may wish to use block level elements such as a blockquote:

>>>
  Once upon a time...
>>>

or maybe even some pre-formatted text:

```
  Pre-formatted text may be used for code
as white-space is preserved.
```

As you can see above, these are described by using 3 symbols at the start of a 
line and must be preceeded and followed by a blank line.  For a full list of
elements see your specific configuration.

Inline formatting & Symbols

Importantly there are also some inline elements that can be useful for 
expressing the true meaning of some words or phrases in your block-level 
paragraphs.

These can be used for ///emphasis///, or giving a phrase a **strong --effect-- 
++emphasis++**.

In addition to these elements some symbols may also be converted for convenience
such as the copyright (c) symbol.

Lists

Another important feature of documents is the list. Creating lists should be apparent from the source-code. The next level is reached by repeating the list symbol.

 * List item 1
 ** Items must have at least 1
    space before the *-symbol.
 * Clearly you can also continue the list item
onto another line if you need to.

 * A blank line will:
 *# end the list
 *# while a #-symbol will
 *## start an ordered-list

Links & Images

Links and images are both similar by design. An opening and closing pair of symbols encloses all relevant details about the link.

A paragraph can contain a [[link:http://google.co.uk "clickable phrase"]] known 
as a hyperlink. or it could contain an image [[img:lib/icons/images.png "images"]].

A paragraph can contain a clickable phrase known as a hyperlink. or it could contain an image .
